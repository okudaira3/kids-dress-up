# 指示書：お着替え遊びWebアプリ用の画像素材加工

## 目的

画像生成AIで作成したお着替え遊び用のイラスト画像を、React製Webアプリで扱いやすい素材に加工する。

最初の目的は以下。

1. 生成済みのカード画像から、各選択肢を個別画像として切り出す
2. Webアプリの選択肢カードに使いやすいサイズへ整える
3. ファイル名をアプリ側で扱いやすい英数字に統一する
4. 可能であれば背景を透明化した「着せ替えプレビュー用素材」も試作する

ただし、着せ替えプレビュー用の完全な透過レイヤー化は難しい可能性があるため、まずは選択肢カード用画像の切り出しを優先する。

## 切り出し方針

まずは自動検出ではなく、座標ベースの切り出しでよい。

理由：
- 生成画像はカード配置がほぼ固定
- 自動検出は誤検出のリスクがある
- MVPでは安定性を優先したい

画像サイズは実際に読み取って確認すること。  
想定サイズと異なる場合は、画像サイズに対する比率で切り出すこと。

## カード画像の出力仕様

切り出した画像は、選択肢カードとしてそのまま使えるようにする。

- PNG形式
- 正方形または4:3寄りでよい
- 余白は残してよい
- 枠線と文字は残してよい
- 横幅は原則 512px にリサイズ
- アスペクト比は維持
- 画質を大きく劣化させない

## 実装方法

Node.js ではなく Python で実装してよい。  
Pillow を使うこと。

作成するスクリプト：

```text
scripts/process_assets.py
```

実行コマンド：

```bash
python scripts/process_assets.py
```

## process_assets.py の要件

- Pillow を使う
- 入力画像が存在しない場合は分かりやすいエラーを出す
- 出力ディレクトリがなければ作成する
- 各画像のサイズを読み取る
- 比率ベースで切り出し範囲を計算する
- 各カード画像を切り出す
- 横幅512pxにリサイズする
- PNGで保存する
- 処理結果をコンソールに表示する

## 比率ベース切り出しの例

厳密な座標ではなく、画像全体の幅・高さに対する割合で指定する。

例：

```python
# left, top, right, bottom を 0.0〜1.0 の比率で指定
crop_ratio = (0.03, 0.04, 0.49, 0.96)
```

実際の画像サイズに変換する。

```python
left = int(width * crop_ratio[0])
top = int(height * crop_ratio[1])
right = int(width * crop_ratio[2])
bottom = int(height * crop_ratio[3])
```

### プレビュー用素材の方針

- 枠線と文字を除去する
- 白背景を可能なら透明化する
- アイテム部分だけを残す
- 透過PNGとして保存する
- 出力サイズは全て 600x800 に統一する
- 位置合わせは仮でよい

## 注意：プレビュー用素材の限界

現在の生成画像は「選択肢カード用」のため、着せ替えレイヤーとして完全には使えない可能性が高い。

理由：
- 服単体画像であり、キャラクターの体型に合わせて描かれていない
- 枠線と文字が入っている
- 余白が多い
- 服のサイズや向きがキャラクターと合わない可能性がある
- リュックなどは背負わせる位置合わせが難しい

そのため、プレビュー用素材はあくまで試作扱いにする。  
本番の着せ替えプレビューは、CSS図形または専用の透過PNG/SVG素材で実装する。

## 品質条件

- `python scripts/process_assets.py` がエラーなく完了すること
- 入力画像がない場合は、どのファイルが足りないか表示すること
- `public/assets/cards/` に13個のカード画像が生成されること
- カード画像に対象アイテムが見切れず含まれていること
- 出力ファイル名がReact側から扱いやすい英数字になっていること
- 画像加工処理は何度実行しても上書きできること

## 実装後に確認すること

1. `python scripts/process_assets.py` を実行
2. `public/assets/cards/` の画像を目視確認
3. `public/assets/debug/contact-sheet.png` を確認
4. 切り出し位置がズレていたら、crop_ratio を微調整
5. Reactアプリの選択肢カード画像を、絵文字から `public/assets/cards/*.png` に差し替える

## 重要

最初から完璧な着せ替えレイヤー素材を作ろうとしないこと。  
まずは選択肢カード用の画像切り出しを確実に成功させること。
